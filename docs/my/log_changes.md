## Лог изменений сервиса 
### Версия 1.0.7

- **Более устойчивое реагирование к перебиваниям**  
  Ассистент стал менее чувствителен к случайным шумам и коротким звукам.  
  Мы понизили чувствительность к перебиванию: теперь он точнее определяет, когда собеседник действительно начал говорить, и реже прерывает свою речь по мелочам.

- **Исправлен баг редактора базы знаний**  
  В редакторе текста базы знаний были проблемы с юзабилити:  
  некорректно удалялись символы, курсор «прыгал» и позиционировался неправильно.  
  Поведение редактора переработано — удаление, перемещение курсора и ввод текста теперь работают ожидаемо и стабильно.

- **Исправлена гонка озвучивания ответов**  
  Ассистент больше не повторяет один и тот же ответ и не отвечает на текущие вопросы фразами, предназначенными для предыдущего запроса.  
  Гонка между записью и воспроизведением устранена, поэтому каждый ответ озвучивается один раз и строго в свою очередь.

- **Новая фича: быстрые ответы ассистента в коннекторах**  
  В настройках коннектора появилась возможность управлять «быстрыми ответами» ассистента — короткими репликами на время, пока основная модель готовит основной ответ.  
  Это делает диалог более живым и непрерывным: пользователь получает ,быстрое подтверждение, что его запрос услышан и обрабатывается.


### Версия 1.0.8

- **Выбор базы знаний для шага диалога**  
  В настройках шага появилась опция *«База знаний для поиска»*. Теперь можно явно указать, из какой базы знаний ассистент будет брать ответы на этом шаге. Поле необязательное — если ничего не выбрано, сценарий работает как раньше.

- **Понятное поведение старта сценария**  
  При первом сообщении в чате ассистент автоматически начинает с первого валидного шага сценария.  
  Текущий шаг сохраняется в контексте чата и затем используется во всех последующих запросах — сценарий «помнит», на каком шаге сейчас находится диалог.

- **Предсказуемый авто‑переход между шагами**  
  Если для шага настроен явный следующий шаг или ветвление по ответам, и все обязательные слоты заполнены, ассистент может автоматически перейти к следующему шагу в рамках одного диалога.  
  Если для шага не настроены ни следующий шаг, ни ветвление, авто‑переходов больше не будет: ассистент останется на этом шаге, пока вы сами не настроите правила перехода.  
  Также **исправлена ошибка**, из‑за которой в некоторых случаях шаг «съезжал» просто на следующий по порядку в списке сценария. Теперь такого некорректного поведения нет.


### Версия 1.0.9+16

- **Привязка коннекторов к сценариям, а не к ассистентам**  
  Логика работы коннекторов переработана: теперь каждый коннектор привязывается не к ассистенту, а к конкретному сценарию диалога. 

- **Выбор сценария диалога в настройках коннектора**  
  В основных настройках коннектора появился выпадающий список со сценариями диалогов. Можно выбрать нужный сценарий по имени.

- **Свитч активации коннектора вместо «подключён к ассистенту»**  
  Переключатель в списке коннекторов теперь управляет полем активности самого коннектора. Старые действия «подключить/отключить коннектор к ассистенту» удалены. Тултип и подпись обновлены: теперь видно, активен коннектор или нет, без привязки к конкретному ассистенту.

- **Чистка устаревшей логики API и провайдеров**  
  Удалены методы API и провайдеры, работавшие с привязкой коннекторов к ассистентам. Клиент теперь использует только единый список коннекторов и их настройки.
